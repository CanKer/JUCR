# Roadmap / Commit Plan

Use small, meaningful commits (Conventional Commits). Each commit should keep the project runnable.

1. `chore: bootstrap project scripts and tsconfig`
2. `chore: add eslint baseline`
3. `chore: add github actions ci pipeline`
4. `chore: add docker-compose mongodb for local dev`
5. `test: add unit tests for poi transformation`
6. `feat: implement poi transformation`
7. `test: add e2e import test with fake ocm server`
8. `feat: implement importer with fetch and mongo upserts`
9. `docs: document local dev, tests and ci`
